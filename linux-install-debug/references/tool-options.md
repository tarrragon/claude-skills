# 工具選項：依環境挑對工具

Linux 上同一件事常有多個工具，預設的不一定最省力。這份給 AI 在執行任務或給建議時挑對工具用，依環境（CLI / 圖形桌面 / 遠端）分。核心判準：**互動 / 人在看的情境用現代替代品，腳本 / 可攜情境用保證存在的 POSIX 工具。**

## CLI 環境：日常指令的現代替代品

| 任務         | 預設   | 現代替代（互動時優先） | 換的理由                             |
| ------------ | ------ | ---------------------- | ------------------------------------ |
| 搜文字內容   | `grep` | `rg`（ripgrep）        | 遞迴 + 尊重 .gitignore + 快 + 帶色   |
| 找檔案       | `find` | `fd`                   | 語法直覺、預設遞迴 / 尊重 .gitignore |
| 互動挑候選   | —      | `fzf`                  | 任何清單 → 模糊選；`Ctrl+R` 歷史搜尋 |
| 看檔案內容   | `cat`  | `bat`                  | 語法高亮、行號、Git 標記             |
| 列目錄       | `ls`   | `eza`                  | 帶色 / 圖示 / Git 欄 / `--tree`      |
| 跳目錄       | `cd`   | `zoxide`（`z`）        | 記住常去目錄                         |
| 看磁碟用量   | `du`   | `dust`                 | 樹狀 + 視覺化                        |
| 簡單文字替換 | `sed`  | `sd`                   | 不用記跳脫規則                       |

**關鍵紀律**：這些是**互動 shell** 的省力工具，不是腳本依賴。寫要在別台機器 / CI 跑的腳本時用 `grep` / `find` / `cat`（保證存在）；現代替代品放使用者的 alias / key binding。判斷「該推薦哪個」時先問：這是使用者互動、還是要寫進腳本？

**搜尋三件套** `rg` / `fd` / `fzf` 投報率最高，優先推薦 / 優先用。

## 圖形桌面：挑 GUI app 看相依足跡

圖形桌面 app 的選擇，成本在相依樹，不在 app 本身。同功能不同實作差一個數量級——有些假設某桌面環境的服務在旁邊（拖進半個 DE），有些刻意做成桌面無關。

檔案管理員為例（實測新裝套件數）：

| 選項       | 出身     | 相依足跡                            | 適用                           |
| ---------- | -------- | ----------------------------------- | ------------------------------ |
| Thunar     | XFCE     | 輕（~8 套件）                       | 最小 WM（如 Hyprland）預設推薦 |
| PCManFM-Qt | LXQt     | 最輕（~7）                          | Qt 棧、想最小相依              |
| Nemo       | Cinnamon | 重（~38，拖 cinnamon-desktop/xapp） | 本來就跑 Cinnamon 才划算       |

判準：最小化 WM 想要圖形 app，優先桌面無關的（Thunar / PCManFM-Qt）；要完整側欄功能（掛載 / 垃圾桶 / 網路）補 `gvfs`、縮圖補 `tumbler`——這些功能各是一個額外相依，「輕量不是免費的」。除非已跑對應 DE，否則別為單一 app 裝整套桌面環境元件。移除拖進 DE 的 app 後記得清孤兒（`pacman -Qtd`）。

## 遠端：把工作留在遠端、斷線不掉

遠端工具圍繞「連線斷了、遠端 session 與長任務還在」。

| 需求                | 工具                        | 備註                             |
| ------------------- | --------------------------- | -------------------------------- |
| session 常駐 / 持久 | `tmux` / `zellij`           | SSH 斷不影響、重連 `attach` 接回 |
| 無人值守長任務      | 多工器 + NOPASSWD + gh auth | 把任務留遠端跑、成果推回         |
| 檔案同步            | `rsync`（單向）/ syncthing  | 增量、可續傳                     |
| 低頻寬 / 手機連線   | 純文字 TUI（不傳影像）      | 監控 / 圖表 / db 都有 TUI 版     |
| 連線更耐斷          | `mosh`（有裝時）            | UDP、漫遊、本地回顯              |

判準：任何要在遠端跑一段時間的東西，先丟進多工器再說——否則 SSH 一斷任務就沒。詳見 [remote-access](remote-access.md)（遠端連線 / 終端機問題的診斷）與 [install-and-verify](install-and-verify.md)（無人值守前置）。

## 推薦工具時的通則

1. **先問環境**：互動 CLI / 圖形桌面 / 遠端——同需求不同環境挑不同工具。
2. **互動 vs 腳本**：互動用現代替代品（省力），腳本用 POSIX（可攜）。
3. **圖形 app 看相依**：推薦前想它拖進什麼，最小環境優先桌面無關的。
4. **遠端優先持久性**：長任務先進多工器。
5. **別假設裝了**：推薦現代工具時附上安裝指令（多在套件庫：`pacman -S ripgrep fd fzf bat eza`）。
