# 審查要有斷言支撐 frame：抓知識類型與分類定位的失配

> **角色**：本卡是 `multi-round-review` 的支撐型原則（principle）、被 Round 1-E 斷言支撐 reviewer 與反模式段引用。
>
> **何時讀**：規劃 reviewer frame 時；或內容通過多輪審查後仍被使用者判定「粗糙 / 像故事 / 深度不足」、要診斷審查框架缺口時。

## 核心原則

多輪審查的 frame 集合要包含「斷言支撐」frame：抽樣每篇承擔判準的核心斷言、問「這個斷言靠什麼成立——機制推導、量化、可驗來源，還是口吻與權威」；並在模組層對照「這個模組的知識類型（分析 / 敘事 / 操作）跟它所在分類的定位與 sibling 的 house style 一致嗎」。

既有的審查 frame——字句層 compliance、cadence、讀者旅程、steelman——都在文字表面與結構層操作。一個模組可以每一句都合規、每個結構都到位，整體卻是錯的知識類型：結論都在、支撐全缺的經驗談，穿著教學結構的衣服全數通過。這是 frame 覆蓋的 design gap：輪數與 keyword 都補不上、要補的是一個新 frame。

## 兩層操作

| 層級   | 檢查問題                                                            | 失配訊號                                       |
| ------ | ------------------------------------------------------------------- | ---------------------------------------------- |
| 斷言層 | 抽 3-5 個承擔判準的斷言，各自的支撐是機制 / 量化 / 來源、還是口吻？ | 支撐是「厲害的人都這樣」「經驗上如此」         |
| 模組層 | 模組的知識類型跟分類定位、sibling house style 一致嗎？              | 分類的分析語言（算式、結構分析）模組完全用不上 |

斷言支撐跟 steelman 的「數字 / 閾值有無源頭」是兩個層：steelman 在句子層看誠實度（「約七成」有標約數、放行）；斷言支撐在判準層看可推導性（「七成的推導在哪、讀者換情境能重算嗎」）。同一句話、兩個 frame、兩種結論——斷言支撐是獨立維度、被 steelman 涵蓋不了。

## 排程判準

- 素材來源是經驗談 / 訪談 / 口述整理的 batch 為高風險——此 frame 排第一輪。知識類型錯位的修法是重寫、越晚抓、已投入的字句層打磨越多全作廢。
- 歷輪 finding 全部落在字句與結構層、沒有任何 finding 質疑斷言站不站得住——分佈本身是「深層無人在看」的訊號、下一輪排此 frame。

## 邊界

- 抽樣即可（每篇 3-5 個承擔判準的斷言）、逐句要 citation 會把審查變成論文評審。
- 支撐類型依知識目標判定：操作型文章的合法支撐是實機驗證、分析型是機制與量化、案例型是可回溯來源——frame 檢查「有沒有支撐」與「類型跟知識目標對不對」、統一形態要求是誤用。
- 心態 / 生態類內容的價值本來就是經驗分享——正確處理是標明知識類型並跟分析內容分離（路由問題）、此 frame 不適用。
