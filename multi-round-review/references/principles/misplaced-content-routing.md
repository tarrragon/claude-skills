# SRP 違反是路由訊號、不是刪除訊號

> **角色**：本卡是 `multi-round-review` 的支撐型原則、被 reviewer frame 設計引用。
>
> **何時讀**：reviewer 標記「這段不屬於這篇」時、決定怎麼處理這個 finding。

## 結論

Review 發現文章裡有不屬於當前主題的段落時，這個段落的存在是路由訊號——有一個議題需要被安置在正確的位置，而不是有多餘的文字需要被清除。

## 為什麼刪除是最差選項

這段內容存在代表作者（或讀者）在某個情境下確實需要這個知識。刪除讓這個需求變成隱性缺口——需求仍在、但沒有文章滿足它。

SRP 違反在文章層面的意義是：這段內容有自己的語意責任、跟文章主線的責任不同。正確回應是幫它找到位置、不是消滅它。

## 處理流程

1. **確認讀者需求存在**：這段內容對應什麼情境下的讀者需求？如果作者當初寫進來、通常有上下文理由（排查時一起看、概念有對比關係）。
2. **找出路**：依成熟度選擇——
   - 已有適合的文章 → 搬過去
   - 值得獨立成篇 → 新開文章
   - 議題還不夠成熟 → 留在原處、用段落標記區隔主題、加 backlog 備註
   - 找不到該去的分類 → 可能缺一個分類（兜底資料夾裡同主題文章群聚是分類浮現的訊號）
3. **reviewer 報告格式**：標記「不屬於」時同時標「建議的目的地」。只標前者不標後者、修改者容易選最省力的動作（刪除）而非最正確的動作（路由）。

## 套用方式

在 reviewer prompt 的審查維度裡加一條：「發現 SRP 違反時、finding 的建議修法欄除了標問題、還要標建議的目的地（已有文章 / 新文章 / 新分類 / 留原處加標記）。」
