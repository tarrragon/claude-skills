# ROI 矩陣防 collapse 雙向誤判

> **角色**：本卡是 `content-extension-evaluation` 的支撐型原則、被「避免兩種反向誤判」段引用。
>
> **何時讀**：判斷「該不該做某個延伸」時、避免靠直覺收尾或無止境疊代。

## 結論

延伸評估的兩個反向誤判分別是「提早收尾」（覆蓋率不足）跟「無止境疊代」（完美主義 collapse）。ROI 矩陣（成本 × 價值 × 急迫度）是兩個誤判的共同防線。

## 兩個誤判的對比

| 誤判       | 機制                                  | 結果                                | 防範                                         |
| ---------- | ------------------------------------- | ----------------------------------- | -------------------------------------------- |
| 提早收尾   | 「核心章節寫完就好」、沒走延伸評估    | 框架已給、執行缺位、讀者撞牆        | 強制 6 軸盤點、確認沒 systemic 缺位          |
| 無止境疊代 | 「全部都做、品質才完整」、低 ROI 也做 | 時間 / context 用光、新章節無法開工 | 修法成本反轉訊號、修一個成本超過讀者價值就停 |

兩個誤判的根因是「沒明示成本 / 價值 / 急迫度三軸」，而是用直覺決定「做哪些」。

## ROI 矩陣的 3 軸

| 軸     | 等級                                                                                    |
| ------ | --------------------------------------------------------------------------------------- |
| 成本   | 低（< 50 行 + 機械修）/ 中（50-300 行）/ 高（300+ 行或重寫）                            |
| 價值   | 低（少數讀者）/ 中（多數讀者次要影響）/ 高（多數讀者核心問題）                          |
| 急迫度 | 立即（核心讀者旅程斷層）/ 短期（影響 < 50% 讀者）/ 長期（覆蓋深度）/ 不補（沒明確訊號） |

每個候選都評三軸、不能跳過。

## 立即補的判讀

立即補的訊號：

- 高價值 + 任何成本 + 急迫度立即 = 立即補（用戶撞到的根本問題）
- 高價值 + 低成本 + 任何急迫度 = 立即補（cheap defense）

## 短期 / 長期 backlog 的判讀

短期 backlog：

- 中價值 + 中成本 + 急迫度短期 = 短期 backlog
- 中價值 + 低成本 + 任何急迫度 = 短期 backlog（順手補）

長期 backlog：

- 中價值 + 高成本 = 長期 backlog（需要規劃時間）
- 低價值 + 任何成本 = 長期 backlog 或不補

## 不補的判讀

- 低價值 + 高成本 = 不補
- 重複度高（已被其他內容涵蓋）= 不補
- 半衰期極短（< 3 個月）= 不補（會立刻過時）

## 套用方式

評估結束時、按 ROI 矩陣輸出表格、用 AskUserQuestion 問用戶要不要按推薦做。讓用戶看到分級而不只看到「N 個候選」、能避免「全做」或「全跳」的兩種 collapse。
