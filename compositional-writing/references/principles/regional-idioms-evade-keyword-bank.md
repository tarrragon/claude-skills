# 地區慣用語直譯：keyword grep 抓不到、同源讀得懂會放行

> **角色**：本卡是 `compositional-writing` 的支撐型原則（principle）、被字句層 keyword bank 的「地區用語」grep 引用、是 [`regional-terminology-alignment`](regional-terminology-alignment.md) 的偵測維度延伸。
>
> **何時讀**：擴充地區用語掃描時、review 回報「地區用語 clean」卻被在地讀者抓到用語時、判斷偵測缺口是 design 還是 execution 時。

---

## 結論

地區用語的偵測分兩層，難度不同。**單詞漂移**（同一概念在不同地區各有一個常用詞，兩兩對應）是封閉、可列舉的集合，用 keyword grep 掃得到（見 [`regional-terminology-alignment`](regional-terminology-alignment.md)）。**慣用語直譯**（把一個抽象動作用生動比喻包裝的口語片語）是開放集合、無法用一份短清單列舉，且每一句「語意讀得懂」，所以列舉式 grep 結構性抓不到、同源審查者也會合理化放行。慣用語直譯的真防線是目標地區讀者的冷讀，不是把 grep 的 keyword 清單加長。

---

## 為什麼慣用語比單詞難抓

### 開放集合 vs 封閉集合

單詞漂移是封閉集合：一個概念在各地區各有一個常用詞，能列成對照表，keyword bank 把對照表做成 grep pattern、掃到就是候選。清單有限、可維護。

慣用語是開放集合：「憑感覺做決定」「可靠」「務實」各可以直譯成好幾個不同地區的生動片語。慣用語的產生是組合性的（動詞 + 比喻 + 語氣），列舉不完。把 grep 清單加長只會追著個案跑、永遠漏掉下一個沒列到的片語——這是 design 上的偵測缺口、不是清單不夠長的 execution 缺口。

### 同源盲區：讀得懂就放行

慣用語直譯的第二層難處是它「可讀」。跨地區訓練語料下，作者跟 LLM 審查者對這類片語都有語感、一讀就懂意思，於是同源審查者把它合理化成「雖然是那邊的說法、但讀者看得懂、可接受」——偵測即使命中，判定也放水。這跟 register 違規（否定起手、喊話、誇飾）的同源自審上限同構（見 [`multi-pass-review-frame-granularity`](multi-pass-review-frame-granularity.md)）：同源審查者跟作者共享文體 / 語料直覺、對這層有結構上限、加再多輪同源 review 都跨不過。

---

## 識別訊號

1. **成語式 / 比喻式的決策與評價詞**：把一個抽象動作用畫面感很強的口語片語包裝、且該片語偏向某個地區的高頻用法。命中訊號是「這句在描述決策 / 評價 / 程度、用了一個很生動的口語片語」。
2. **表面讀得懂、但在地不這樣說**：判準不是「看不看得懂」（都看得懂），是「這個地區的人會不會這樣說」。要判定得問在地語感、不是問語意。
3. **review 回報字句層 clean、卻被在地讀者一眼抓到**：這個落差本身是訊號——clean 是「grep 清單沒中」、不是「在地讀者讀起來對」。同源 reviewer 的 clean 對慣用語層不可當真、要標「未經在地抽查」。

---

## 修法：偵測靠在地冷讀、不是加長清單

1. **機械層只做已知個案**：把已踩過的慣用語加進 keyword bank、讓存量 debt 持續可見、不再依賴記憶。但明確標「本清單是已知個案、不是窮舉」——它抓存量、不保證抓新個案。
2. **判定層交在地讀者冷讀**：新個案的真防線是作者以外、且是目標地區的眼睛。這跟 register 違規的異源複核是同一套操作——同源負責曝光候選、在地異源負責定奪。同源 review 對慣用語層的產出是「給在地人複核的清單」、不是「已複核乾淨」。

---

## 何時不算違規

- 引用對方地區的原文 / quote：保留準確性比對齊重要。
- 描述對方地區的具體社會現象本身：不是拿它當一般形容詞。
- 已滲透到在地主流語料的詞：判定要跟著讀者群更新。
- 個人風格 / 訪談保留：narrative 選擇。

判準跟 [`regional-terminology-alignment`](regional-terminology-alignment.md) 一致：問「讀者地區是否單一、這個片語是不是讓在地讀者覺得出戲」、而不是「看不看得懂」。

## 判讀徵兆

| 訊號                                      | 該做的行動                                                     |
| ----------------------------------------- | -------------------------------------------------------------- |
| 出現成語式 / 比喻式的地區口語決策評價片語 | 換成在地說法                                                   |
| 想靠「加長 grep 清單」把慣用語掃乾淨      | 停——這是開放集合、清單追不完；已知個案入清單、新個案交在地冷讀 |
| 同源 reviewer 回報「地區用語 clean」      | 標「未經在地抽查」、慣用語層不當真                             |
| 分不清偵測缺口是 design 還是 execution    | 單詞漏抓 = execution（補清單）；慣用語漏抓 = design（要異源）  |

**核心原則**：地區用語的偵測、單詞層可機械化、慣用語層不可。把慣用語當成「加長清單就能解」是把 design 缺口誤當 execution 缺口——它的真防線是在地讀者的異源冷讀。
