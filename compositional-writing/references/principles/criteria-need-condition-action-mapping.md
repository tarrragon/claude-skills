# 判準要寫到條件層：維度清單是判準的空殼

> **角色**：本卡是 `compositional-writing` 的支撐型原則（principle）、被 SKILL.md 原則 3 的「判準寫到條件層」段引用。
>
> **何時讀**：寫教學文章的判準段時；或審查發現某段「列了要考慮什麼、卻推不出該做什麼」時。

## 核心原則

教學文章的判準要寫到條件層：條件到行動的映射，加上映射失效的情境。判準有三個成熟度：

| 成熟度   | 形態                                     | 讀者拿到什麼                   |
| -------- | ---------------------------------------- | ------------------------------ |
| 口訣     | 無推導的結論（「不超過某個比例」）       | 一條要背的規則、換情境即失效   |
| 維度清單 | 「判斷看 A、B、C」                       | 知道要考慮什麼、推不出該做什麼 |
| 條件映射 | 「A 成立 → 做 X；A 破 → 切 Y」＋失效情境 | 帶自己的參數能得出行動         |

維度清單最有欺騙性：它有判準動詞（「判斷」「取決於」）、每個維度都有機制支撐、通過字句層與機制層的審查——卻在讀者要做決定的那一刻斷線。它是判準的空殼，而且機制重建完成之後仍會殘留（機制正確與判準到位是兩個獨立的檢查）。

驗收用**重算測試**：讀者帶自己的參數走進這段判準，能不能走出一個行動。走出來的是「我需要再想想」，這段還停在第二層。

## 條件不可窮舉時的合法變體

自查問句組：把決策變數轉成讀者答得出來的問題，加一條排序或行動規則。它跟維度清單的差別在可答性——維度清單給名詞（「訂單信用」）、自查問句給可自答的問題（「過去幾季的承諾兌現率多高」）。

## 判讀徵兆

- 判準句的動詞停在「看 / 考慮 / 取決於」、沒有「→ 就」的映射——補條件映射。
- 讀者問「所以我該怎麼做」答不出來——重算測試失敗。
- 判準段沒有失效情境——補「條件破了切到哪」。
- 批次稿件的判準段句式雷同（「判斷看 A、B、C」連發）——空殼正被當模板複製，逐段跑重算測試。

## 邊界

背景與機制段不承擔行動交付、不適用本卡；維度清單作為條件映射的前導（先列維度、再給映射）合規，違規的是列完就收尾；探索型筆記記錄未收斂的思考、不強制到第三層，但要標明知識類型。
