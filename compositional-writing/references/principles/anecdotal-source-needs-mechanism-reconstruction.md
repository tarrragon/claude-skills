# 經驗談來源要重建分析層、不是換敘述口吻

> **角色**：本卡是 `compositional-writing` 的支撐型原則（principle）、被 `source-to-teaching-analysis.md` 的機制重建 pass 引用。
>
> **何時讀**：把從業者經驗談（訪談、社群貼文、口述整理）轉成教學文章時；或改寫稿被判定「像講故事、深度不足」、要診斷轉換哪裡沒做到位時。

## 核心原則

經驗談的斷言是從業者壓縮過的結論——「不超過七成」「絕不允許 X」「該備多久看交期」都是有效經驗，但推導路徑留在從業者腦中，素材本身沒有自帶分析層。教學轉換的核心工作是重建這條路徑：用成本結構、誘因結構、市場結構把斷言還原成可推導、可質疑、可遷移的機制，讓每個承擔判準責任的斷言能回答「為什麼成立、在什麼條件下不成立」。

只做系統化整理——分主題、換書面語、加表格、補術語卡——會產出「披著教學結構的經驗談」：結構合規、字句合規、結論都在、支撐全缺。這種產物通得過字句層與結構層審查，但讀者拿到的是「要相信的規則」，不是「能自己推的判斷」；情境一變（比例、產業、規模），規則失效，讀者沒有機制可以重算。

## 跟「分析文 source」的分工

分析文（分析師文章、備忘錄）自帶分析層，轉換工作是**拆層**——分開事實、原作者判讀、本文推導，防止原作者 frame 被當成事實。經驗談只有事實與判讀兩層，分析層要**重建**。判別訊號看斷言的支撐：分析文的斷言後面有數據與機制，經驗談的斷言後面是經驗本身（「做這行都知道」）。

## 三層分工

| source 內容層                          | 處理方式                                    |
| -------------------------------------- | ------------------------------------------- |
| 事實（規則條款、流程、事件）           | 保留，構成教材的 fact 層                    |
| 從業者判讀（閾值、口訣、「該這樣做」） | 當 hypothesis，重建支撐機制後才升為教材判準 |
| 分析層（機制、量化、結構解釋）         | source 沒有自帶，由作者從領域通用知識補建   |

## 判讀徵兆

- 教材斷言的支撐是「厲害的 / 資深的人都這樣做」——訴諸權威即知識類型未升級，重建機制。
- 數字閾值無推導——找出閾值是什麼約束的反推值，把數字降級為機制的例示。
- 同一句 anecdote 出現在多篇——素材未消化，找出它支撐的機制、收斂到一處。
- 規則被描述、但「規則為什麼存在」缺席——補設計者的誘因分析。

## 邊界

重建有停止線：只有承擔判準責任的斷言需要機制，氛圍性敘述刪除或降為 hook。心態與職場修為內容是合法的經驗分享，但知識類型是 career essay——分開成篇並標明定位，處理方式是路由不是重建。兩條重建路徑（領域通用理論 / 回頭追問從業者）都走不通的斷言，標明「經驗值、機制待補」，別偽裝成已推導。
