# 會誤解處補正向知識、不把誤會當敘事中心

> **角色**：本卡是 `compositional-writing` 的支撐型原則（principle）、被 SKILL.md 字句層 keyword bank（澄清式框架 grep）+ multi-pass review 輪 8-9 引用。
>
> **何時讀**：寫技術教材 / 概念解釋 / 實作記錄時、發現段落用「這是最容易誤解的一點 / 常見的誤判 / 要抵抗…的直覺 / 值得記 / 會困惑的現象」這類把「讀者會誤解」當敘事中心的起手時；尤其在 multi-pass review 跑 keyword bank 比對、以及 reader-simulation 語意 pass。

---

## 論述基礎與限制

本卡的論述基於 **1 個 batch**（一個技術教材批次的三輪 review、一次抓到 6 處同構的澄清式框架）抽出、建卡判準是教學需求（這類 framing 在技術寫作、尤其 LLM 生成內容裡高發）。具體限制：

- **命中詞是 starting set、不窮舉**：新變體（「你以為 X 其實 Y」「乍看…實際」「反直覺」）出現時持續擴充；同義變體多、grep 抓不全、要靠語意判定。
- **界線是本卡的核心、不是附註**：具體第一人稱實測敘事跟真實診斷區分不是違反 —— 誤判會把正向診斷知識一起砍掉。判定靠「概念位置」不靠「有沒有提到誤判」。
- **修補有效性未獨立驗證**：改成正向先行後讀起來更聚焦、但跟讀者實際理解的相關性沒做使用者測試。

---

## 核心原則

**文章與個別知識的目的是釐清觀念、把知識串起來讓讀者融會貫通。** 若某處會讓人誤解、正解是補上讓誤解無從發生的正向知識（先給正確心智模型與支撐它的機制），不是喊「這容易誤會 / 別搞混 / 你以為的其實不對」。

關鍵的認知轉換：**「這容易誤會」是一個訊號、標示作者這裡沒把機制講夠 —— 它不是一段該寫進文章的澄清。** 補救是把知識補到位讓警告沒必要存在、而不是把警告寫得更清楚。把 misconception 放在正確模型之前、先把讀者放進「你會錯」的位置、還常提醒讀者一個他本來不會有的困惑 —— 目標該是讓讀者融會貫通、不是提醒他「不需要特別認知的內容」。

| 澄清式框架的形式        | 句型                                   | framing 的問題                       | 正向改法                                 |
| ----------------------- | -------------------------------------- | ------------------------------------ | ---------------------------------------- |
| meta 標記「最容易誤解」 | 「這是最容易誤解的一點」               | 把誤解重新推上台面、不傳遞知識       | 刪 meta、讓前面已給的正確模型自己站住    |
| 誤解當主題句起手        | 「要點破一個會誤解的心智模型：…」      | 正確模型被冒號擠到後面               | 機制直述先行：「這個模型的直接後果是…」  |
| 錯誤信念當句子主詞      | 「『X 應該安全』是常見的誤判」         | 先立誤解再打掉、正確知識被推到第二句 | 直述正確知識、讓誤解無從發生             |
| 喊話式抵抗直覺          | 「選型時要抵抗『功能多所以好』的直覺」 | 以讀者誤判當敘事骨架、兼帶喊話       | 給正向判準：「判準是這個場景會不會用到」 |

## 界線：哪些不改（本卡的核心判準）

誤判界線會把正向診斷知識一起砍掉、所以界線比命中詞更重要。判別線是「這處在**補正確模型**、還是**把假想誤會當敘事中心**」：

- **具體第一人稱實測敘事**（保留）：「一度以為某個鍵缺、實際是鍵列可捲動被遮住」是真實經歷的教訓、教的是「以互動為準」的判讀、不是提醒讀者一個假想困惑。事件敘事 / 實作記錄以直覺作對照合理。
- **真實診斷區分**（保留）：「連線逾時 vs 連線被拒」「症狀層 vs 根因層」是真實存在的技術事實、教它們就是「補上讓讀者理解的正向知識」。以正向方式供給（這是什麼、怎麼判別、機制為何）、提到常見誤判只作對照不主導、即合規。
- **把假想誤會當主題句起手**（改）：讀者本來不會有這個困惑、文章卻先植入再打掉。

## keyword bank + 判定

```text
澄清式框架：
  最容易誤 / 容易誤判 / 常見的誤判 / 要點破 / 直覺帶偏 / 抵抗…的直覺 / 你以為 / 會困惑 / 值得記
```

**命中是候選、不是判決**：grep 命中後要一個語意判定步驟——這處在「補正向知識」還是「提醒讀者會犯錯」。自問「拿掉這個誤會 framing、正確知識還在不在」：在 → 刪 framing；不在 → 這處根本沒給知識、要補的是機制不是警告。register 類同義變體多、無穩定關鍵詞、reader-simulation 語意 pass 是主、keyword bank 是輔。判定層的完整論述見 [multi-pass-review-frame-granularity](multi-pass-review-frame-granularity.md)。

## 跟其他原則的關係

- [teaching-prose-neutral-register](teaching-prose-neutral-register.md)：兩卡相鄰但軸不同。該卡是 **stance** 軸（陳述概念 vs 對讀者說話）、把「別搞混」當祈使喊話的一例、修法是刪 stance。本卡是 **知識供給** 軸：核心是「『容易誤會』標示知識缺口、要補正向模型」、修法是刪 framing + 補 / 確認機制。同一個句子、兩個維度。
- [multi-pass-review-frame-granularity](multi-pass-review-frame-granularity.md)：本卡的命中同屬該卡 keyword bank 機制的類別；「命中是候選不是判決」「register 類靠 reader-simulation 不靠 keyword」的判定層論述以該卡為主。
- [teach-judgment-not-procedure](teach-judgment-not-procedure.md)：「這容易誤會」是壓縮結論的一種 —— 作者已走完「讀者需要 X 知識才不會誤解」的推導、只輸出最後一步的警告、丟掉了 X。本卡的修法（補回正向模型）正是還原推導路徑。
