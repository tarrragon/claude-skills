# 原子筆記要有向上的議題入口：承載知識、不是字典

> **角色**：本卡是 `compositional-writing` 的支撐型原則（principle）、被 SKILL.md 原則區引用，補「原子筆記的進入動機」維度（既有原則多談一篇之內的字句 / 結構，本卡談一篇之上的入口）。
>
> **何時讀**：寫 Zettelkasten 卡、glossary、術語條目、任何「可被搜尋或直連單獨抵達的原子筆記」時，用本卡判斷它有沒有給讀者「為何讀這張」的進入動機。
>
> **來源**：blog report #169（til/terms 14 卡撰寫檢討）。

## 論述基礎與限制

本卡抽自 1 個 case（til/terms 一組跨領域術語卡）。限制：單一案例、單一內容類型（術語卡）；「議題 hub + 原子卡」兩層結構對其他內容類型的適用性未跨案例驗證。與 reader-level / teaching 旅程類原則的差別：那些談一篇之內的讀者層級或課程級學習路線，本卡談**單張原子筆記之上的入口層**。

讀者使用本卡時，先判斷內容是否屬「承載知識的原子筆記」（卡片 / glossary / 術語）——是 → 套用；否（一次性筆記、草稿、純查閱表）→ 放寬。

## 核心原則

**承載知識的原子筆記不是字典條目。** 字典回答「這個詞是什麼」；承載知識的卡要回答「**你在討論什麼、撞到什麼問題，才需要這個知識**」。撰寫者有預設的情境與前提，讀者沒有——所以每張原子卡（或它的上層）要提供進入動機：從情境進入，而非劈頭給定義。

具體做法是建一層**議題入口（hub）**：以讀者會遇到的問題為標題（「測試紅燈不一定是真的壞」「告警太多反而沒人看」），討論議題、再分流到底下的原子卡；原子卡頂部回指它出自哪個議題，讓搜尋直接落地者也有回路。形成「議題 hub（情境）→ 原子卡（定義）」兩層，讀者由動機而非定義進入。

| 反模式 | 徵兆 | 修法 |
|--------|------|------|
| 卡淪為字典 | 開頭是「X 指……」定義句、無情境 | 卡頂加一行「這個詞出現在『[議題]』這個問題裡」回指議題 hub |
| 缺進入動機 | 問「讀者為何讀這張」答不出、或答案是「為了知道定義」 | 上層建議題 hub，從讀者遇到的問題進入、分流到卡 |
| 一組互連卡無入口 | 卡彼此互連、卻沒有上層議題導讀 | 為每個分支建議題 hub 當入口 |
