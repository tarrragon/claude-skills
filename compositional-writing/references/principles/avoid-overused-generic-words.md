# 避免濫用泛用詞：具體詞讓文章更深刻

> **角色**：本卡是 `compositional-writing` 的支撐型原則（principle）、被字句層 keyword bank（輪 8）與 multi-round-review Round 1-A 引用。
>
> **何時讀**：寫作 audit 掃字句層時、或發現一個泛用詞（坑 / 東西 / 搞 / 處理）在一篇裡反覆出現時。

## 核心原則

一個泛用詞反覆使用，會把本來不同的情境壓成同一個模糊標籤。每個情境有它自己精確的詞——把「坑」拆開：非預期的行為是**意外**、埋著等你踩的前提是**陷阱**、當下的故障是**出問題 / 故障**、狀態異常是**發生狀況**。用精確的詞，讀者對每個情境的印象清楚且各自獨立；用同一個泛用詞蓋過去，這些情境在讀者腦中糊成一片。詞彙的變化本身是資訊——它告訴讀者「這幾件事不一樣」。

## 為什麼泛用詞會被濫用

- **省思考**：泛用詞什麼都能套，用它就不必停下來想「這個情境到底是哪一種」。精確詞逼你先分類。
- **LLM 自產傾向**：模型高頻詞（坑 / 東西 / 搞 / 處理）填空成本低、順手就出來。
- **網路語感滲透**：「踩坑 / 避坑」在部分中文技術社群是高頻詞、另一些地區原本少用，讀多了會不自覺帶進來。
- **用順了就整篇重複**：這是句型 cadence 同質化在字詞層的版本——一個詞用順了，整篇都是它。

## 「坑」的替代詞（依情境選）

| 情境                    | 別用「坑」、用   |
| ----------------------- | ---------------- |
| 非預期的行為 / 結果     | 意外、非預期行為 |
| 埋著等人踩的設計 / 前提 | 陷阱、地雷       |
| 當下的故障              | 出問題、故障     |
| 狀態異常                | 發生狀況、異常   |
| 難處理的細節            | 眉角、難處、細節 |

判準不是背這張表，是先問「這裡具體是什麼」——答案出來，精確的詞自然浮現。

## 更廣：被濫用的泛用詞不只「坑」

同一類還有「東西」「搞」「弄」「處理」「做」「情況」。共同特徵是：一個泛用詞在一段或一篇裡反覆出現、而每次指的其實是不同的具體事。判準不是「這個詞壞」，是「同一個泛用詞被拿來蓋過本該分開的具體情境」。看到密集重複就回頭問：這裡具體是什麼，換成那個具體詞。

## keyword bank grep 與判定

```text
rg "坑|東西|搞|弄|處理一下|情況" <files>
```

命中是候選、不是判決（同 keyword bank 通則）。合規形態：真的就是泛指、沒有更精確的詞；引號內引用原話；口語 hook / narrative 輕度使用且不主導段落。違規形態：命中密集、且每次指的是不同的具體事。判定看「這個詞蓋掉了本該分開的具體情境嗎」。

## 判讀徵兆

- 一篇文章「坑」出現 3 次以上、每次指不同的事。
- 讀完記不清「那些坑」各自是什麼——被壓成同一個模糊印象。
- grep 命中密集且集中在同一批段落。
